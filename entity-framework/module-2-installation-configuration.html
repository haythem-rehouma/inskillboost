Installation et Configuration

Prérequis et Outils Nécessaires (Guide Complet pour Débutants)

Avant de commencer à utiliser Entity Framework, nous devons nous assurer que votre ordinateur dispose de tous les outils nécessaires. C'est comme préparer sa cuisine avant de cuisiner !

Qu'est-ce qu'on va installer et pourquoi ?

Analogie simple

Imaginez que vous voulez construire une maison. Vous avez besoin :

  • D'outils (marteau, scie, etc.) → .NET SDK
  • D'un plan de travail (établi) → IDE (Visual Studio, VS Code)
  • D'un endroit pour stocker vos matériaux (garage) → Base de données

Entity Framework a besoin de ces mêmes éléments pour fonctionner !

1. Le .NET SDK - Votre "Boîte à Outils"

Qu'est-ce que le .NET SDK ?

Le SDK (Software Development Kit) est comme une boîte à outils géante qui contient :

  • Le compilateur : transforme votre code C# en programme exécutable
  • Les bibliothèques de base : fonctions prêtes à utiliser
  • Les outils de ligne de commande : pour créer, compiler et exécuter vos projets
  • Le runtime : l'environnement pour exécuter vos applications

Analogie : C'est comme avoir tous les outils d'un mécanicien dans une seule valise !

Comment télécharger .NET ?
  1. Allez sur https://dotnet.microsoft.com/download
  2. Téléchargez .NET 8 SDK (version LTS recommandée)
  3. Lancez l'installateur et suivez les instructions
  4. Redémarrez votre ordinateur après l'installation

3. Une Base de Données - Votre "Entrepôt de Données"

Pourquoi avons-nous besoin d'une base de données ?

Entity Framework est un pont entre votre code C# et une base de données. Sans base de données, c'est comme avoir un pont qui ne mène nulle part !

Analogie : Si votre code C# est une voiture, la base de données est le garage où vous stockez toutes vos affaires.

SQLite (Recommandé pour débuter)
  • Aucune installation requise
  • Base de données dans un simple fichier
  • Parfait pour apprendre et tester
  • Utilisé par de nombreuses applications mobiles

Parfait pour : Apprentissage, prototypes, applications simples

SQL Server (Pour la production)
  • Base de données professionnelle de Microsoft
  • Très puissante et robuste
  • Excellente intégration avec .NET
  • Version gratuite : SQL Server Express

Parfait pour : Applications professionnelles, gros volumes de données

Notre stratégie d'apprentissage

Dans cette formation, nous commencerons avec SQLite car :

  • Pas d'installation compliquée
  • Pas de configuration serveur
  • Vous pouvez vous concentrer sur Entity Framework, pas sur la base de données
  • Plus tard, nous verrons comment passer à SQL Server

ÉTAPE 2 : Installation des Packages Entity Framework

Qu'est-ce qu'un package NuGet ?

Un package est comme une boîte qui contient du code prêt à utiliser, créé par d'autres développeurs. NuGet est le "magasin" où .NET stocke tous ces packages.

Analogie : C'est comme télécharger une app sur votre téléphone, mais pour ajouter des fonctionnalités à votre projet de programmation !

1
Installer le package principal Entity Framework
dotnet add package Microsoft.EntityFrameworkCore
Que fait cette commande ?
  • dotnet add package → Ajoute un package à votre projet
  • Microsoft.EntityFrameworkCore → Le nom du package EF principal
  • Cette commande télécharge et installe Entity Framework
  • Vous verrez des messages de téléchargement défiler
Patience !

La première installation peut prendre 1-2 minutes car .NET télécharge tous les fichiers nécessaires. C'est normal !

2
Installer le provider SQLite (base de données simple)
dotnet add package Microsoft.EntityFrameworkCore.Sqlite
Qu'est-ce qu'un provider ?

Un provider est comme un "adaptateur" qui permet à Entity Framework de parler à un type spécifique de base de données.

  • SQLite provider → Pour les bases de données SQLite
  • SQL Server provider → Pour les bases de données SQL Server
  • PostgreSQL provider → Pour les bases de données PostgreSQL
  • Nous utilisons SQLite car c'est le plus simple pour apprendre !
3
Installer les outils de développement (optionnel mais recommandé)
Outils Entity Framework :
dotnet add package Microsoft.EntityFrameworkCore.Tools
Outils de design :
dotnet add package Microsoft.EntityFrameworkCore.Design
À quoi servent ces outils ?
  • Tools → Commandes pour gérer la base de données (migrations, etc.)
  • Design → Outils pour générer du code automatiquement
  • Ces outils nous seront utiles dans les prochains modules
  • Mieux vaut les installer maintenant !

ÉTAPE 3 : Vérification de l'Installation

1
Vérifier les packages installés

> Microsoft.EntityFrameworkCore.Tools  8.x.x

Parfait ! Si vous voyez ces 4 packages, l'installation a réussi.

2
Vérifier que le projet compile
dotnet build
Que fait cette commande ?
  • dotnet build → Compile (construit) votre projet
  • Vérifie qu'il n'y a pas d'erreurs dans le code
  • Télécharge toutes les dépendances si nécessaire
Résultat attendu :
Build succeeded.
    0 Warning(s)
    0 Error(s)

Si vous voyez "Build succeeded", c'est parfait !

Que faire si ça ne marche pas ?
Problème 1 : "dotnet command not found"
  • → .NET SDK n'est pas installé ou pas dans le PATH
  • → Redémarrez votre terminal après l'installation de .NET
  • → Vérifiez avec dotnet --version
Problème 2 : Erreurs de packages
  • → Vérifiez votre connexion internet
  • → Essayez : dotnet restore
  • → Supprimez le dossier "bin" et "obj" puis refaites dotnet build
Problème 3 : Versions incompatibles
  • → Utilisez toujours la même version pour tous les packages EF
  • → Spécifiez une version : dotnet add package Microsoft.EntityFrameworkCore --version 8.0.0
Félicitations !

Si vous êtes arrivé jusqu'ici sans erreur, vous avez installé Entity Framework Core avec succès ! Votre environnement de développement est maintenant prêt pour la suite de la formation.

Configuration de Base

1. Création du modèle de données

// Models/Product.cs
using System.ComponentModel.DataAnnotations;

namespace EFCoreDemo.Models
{
    public class Product
    {
        public int Id { get; set; }
        
        [Required]
        [MaxLength(100)]
        public string Name { get; set; }
        
        [MaxLength(500)]
        public string Description { get; set; }
        
        public decimal Price { get; set; }
        
        public DateTime CreatedDate { get; set; } = DateTime.Now;
        
        public bool IsActive { get; set; } = true;
    }
}

2. Création du DbContext

// Data/ApplicationDbContext.cs
using Microsoft.EntityFrameworkCore;
using EFCoreDemo.Models;

namespace EFCoreDemo.Data
{
    public class ApplicationDbContext : DbContext
    {
        public DbSet<Product> Products { get; set; }

        public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options)
            : base(options)
        {
        }

        protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
        {
            if (!optionsBuilder.IsConfigured)
            {
                optionsBuilder.UseSqlServer(
                    "Server=(localdb)\\mssqllocaldb;Database=EFCoreDemoDB;Trusted_Connection=true;MultipleActiveResultSets=true");
            }
        }

        protected override void OnModelCreating(ModelBuilder modelBuilder)
        {
            modelBuilder.Entity<Product>(entity =>
            {
                entity.HasKey(e => e.Id);
                entity.Property(e => e.Name).IsRequired().HasMaxLength(100);
                entity.Property(e => e.Description).HasMaxLength(500);
                entity.Property(e => e.Price).HasColumnType("decimal(18,2)");
            });
        }
    }
}

3. Configuration avec Dependency Injection (ASP.NET Core)

// Program.cs (ASP.NET Core)
using Microsoft.EntityFrameworkCore;
using EFCoreDemo.Data;

var builder = WebApplication.CreateBuilder(args);

// Configuration de la chaîne de connexion
builder.Services.AddDbContext<ApplicationDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("DefaultConnection")));

var app = builder.Build();

app.Run();

4. Fichier de configuration appsettings.json

{
  "ConnectionStrings": {
    "DefaultConnection": "Server=(localdb)\\mssqllocaldb;Database=EFCoreDemoDB;Trusted_Connection=true;MultipleActiveResultSets=true"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.EntityFrameworkCore": "Information"
    }
  }
}

Premier Test de Fonctionnement 

// Program.cs
using Microsoft.EntityFrameworkCore;
using EFCoreDemo.Data;
using EFCoreDemo.Models;

using var context = new ApplicationDbContext();

// Créer la base de données si elle n'existe pas
await context.Database.EnsureCreatedAsync();

// Ajouter un produit de test
var product = new Product
{
    Name = "Ordinateur portable",
    Description = "PC portable haute performance",
    Price = 1299.99m
};

context.Products.Add(product);
await context.SaveChangesAsync();

// Récupérer tous les produits
var products = await context.Products.ToListAsync();

foreach (var p in products)
{
    Console.WriteLine($"Produit: {p.Name} - Prix: {p.Price:C}");
}

Console.WriteLine("Entity Framework Core fonctionne correctement !");

Exercice 3 : Premier Code Entity Framework (Intermédiaire)

Objectif : Écrire votre premier code avec Entity Framework

Ce que nous allons faire :

Nous allons créer une classe simple Livre et un DbContext pour gérer notre bibliothèque. Ne vous inquiétez pas si vous ne comprenez pas tout, nous détaillerons dans les prochains modules !

1
Créer la classe Livre

Créez un nouveau fichier Livre.cs dans votre projet :

// Livre.cs
using System;

namespace MaBibliotheque
{
    public class Livre
    {
        // Propriétés de base d'un livre
        public int Id { get; set; }                    // Identifiant unique
        public string Titre { get; set; }              // Titre du livre
        public string Auteur { get; set; }             // Nom de l'auteur
        public DateTime DatePublication { get; set; }   // Date de publication
        public decimal Prix { get; set; }              // Prix du livre
        public bool EstDisponible { get; set; } = true; // Disponible ou emprunté
    }
}
Explication du code :
  • public int Id → Clé primaire automatique
  • public string Titre → Colonne pour le titre
  • = true → Valeur par défaut
  • Chaque propriété deviendra une colonne dans la base
2
Créer le DbContext

Créez un nouveau fichier BibliothequeContext.cs :

// BibliothequeContext.cs
using Microsoft.EntityFrameworkCore;

namespace MaBibliotheque
{
    public class BibliothequeContext : DbContext
    {
        // Cette propriété représente la table "Livres" dans la base
        public DbSet<Livre> Livres { get; set; }

        // Configuration de la connexion à la base de données
        protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
        {
            // Utiliser SQLite avec un fichier local
            optionsBuilder.UseSqlite("Data Source=bibliotheque.db");
        }
    }
}
Explication du code :
  • : DbContext → Notre classe hérite de DbContext
  • DbSet<Livre> Livres → Collection de tous les livres
  • UseSqlite("Data Source=bibliotheque.db") → Base SQLite dans un fichier
  • Le fichier bibliotheque.db sera créé automatiquement
Questions de compréhension :
  1. Où se trouve le fichier de base de données créé ?
  2. Que fait context.Database.EnsureCreatedAsync() ?
  3. Pourquoi utilise-t-on await context.SaveChangesAsync() ?
  4. Si vous relancez le programme, combien de livres aurez-vous ?

Résolution des Problèmes Courants

Erreur de connexion

Symptôme : "Cannot open database"

Solution :

  • Vérifiez la chaîne de connexion
  • Assurez-vous que SQL Server est démarré
  • Vérifiez les permissions
Conflit de versions

Symptôme : "Assembly version conflict"

Solution :

  • Utilisez la même version pour tous les packages EF
  • Nettoyez et reconstruisez le projet
  • Vérifiez les dépendances transitives
Commandes utiles pour le diagnostic
dotnet list package --include-transitive
dotnet clean && dotnet restore
