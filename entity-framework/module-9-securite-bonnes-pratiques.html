Protection contre les Injections SQL CRITIQUE

Qu'est-ce qu'une Injection SQL ?

Le Danger des Injections SQL

L'injection SQL est l'une des vulnérabilités les plus dangereuses. Un attaquant peut :

  • 🔓 Accéder à toutes vos données
  • 🗑️ Supprimer votre base de données
  • 👤 Voler les informations personnelles
  • 💸 Compromettre les données financières

Exemple d'attaque :

-- L'utilisateur saisit : admin'; DROP TABLE Users; --
-- Votre requête devient :
SELECT * FROM Users WHERE Username = 'admin'; DROP TABLE Users; --' AND Password = '...'
-- Résultat : Votre table Users est supprimée ! 💥

Entity Framework : Protection Automatique

Bonne Nouvelle : EF Protège Automatiquement

Entity Framework utilise automatiquement des requêtes paramétrées qui empêchent les injections SQL. Vos requêtes LINQ sont sûres par défaut ! 

// ✅ SÉCURISÉ - EF paramètre automatiquement
string userInput = "admin'; DROP TABLE Users; --";
var user = await context.Users
    .Where(u => u.Username == userInput)  // Paramétré automatiquement
    .FirstOrDefaultAsync();

// SQL généré (sécurisé) :
// SELECT * FROM Users WHERE Username = @p0
// Paramètre @p0 = "admin'; DROP TABLE Users; --"

Cas Dangereux : SQL Brut

DANGER : FromSqlRaw avec Concaténation
// ❌ VULNÉRABLE - Concaténation directe
string userInput = "admin'; DROP TABLE Users; --";
var users = context.Users
    .FromSqlRaw($"SELECT * FROM Users WHERE Username = '{userInput}'")
    .ToList();

// ❌ VULNÉRABLE - Interpolation de chaîne
var users2 = context.Users
    .FromSqlRaw($"SELECT * FROM Users WHERE Username = '{userInput}'")
    .ToList();
SÉCURISÉ : Paramètres avec FromSqlRaw
// ✅ SÉCURISÉ - Paramètres explicites
string userInput = "admin'; DROP TABLE Users; --";
var users = context.Users
    .FromSqlRaw("SELECT * FROM Users WHERE Username = {0}", userInput)
    .ToList();

// ✅ SÉCURISÉ - SqlParameter
var parameter = new SqlParameter("@username", userInput);
var users2 = context.Users
    .FromSqlRaw("SELECT * FROM Users WHERE Username = @username", parameter)
    .ToList();

// ✅ SÉCURISÉ - FromSqlInterpolated (recommandé)
var users3 = context.Users
    .FromSqlInterpolated($"SELECT * FROM Users WHERE Username = {userInput}")
    .ToList();

Checklist Sécurité SQL

À Vérifier dans Votre Code
  • ✅ Utilisez LINQ autant que possible (protection automatique)
  • ✅ Si SQL brut nécessaire, utilisez FromSqlInterpolated
  • ✅ Jamais de concaténation de chaînes pour SQL
  • ✅ Validez et nettoyez les entrées utilisateur
  • ✅ Utilisez des listes blanches pour les valeurs autorisées
  • ✅ Limitez les permissions de la base de données

Gestion des Erreurs et Logging ÉLEVÉ

Types d'Erreurs Entity Framework

Erreurs Courantes à Gérer
  • DbUpdateException → Erreurs de sauvegarde (contraintes, clés dupliquées)
  • DbUpdateConcurrencyException → Conflits de concurrence
  • SqlException → Erreurs de base de données (connexion, timeout)
  • InvalidOperationException → Erreurs de configuration ou d'usage
Gestion Robuste des Erreurs
public async Task<Result<User>> CreateUserAsync(CreateUserRequest request)
{
    try
    {
        var user = new User
        {
            Username = request.Username,
            Email = request.Email,
            CreatedAt = DateTime.UtcNow
        };

        context.Users.Add(user);
        await context.SaveChangesAsync();
        
        logger.LogInformation("User created successfully: {UserId}", user.Id);
        return Result.Success(user);
    }
    catch (DbUpdateException ex) when (IsUniqueConstraintViolation(ex))
    {
        logger.LogWarning("Duplicate user creation attempt: {Username}", request.Username);
        return Result.Failure("Un utilisateur avec ce nom existe déjà");
    }
    catch (DbUpdateException ex)
    {
        logger.LogError(ex, "Database error while creating user: {Username}", request.Username);
        return Result.Failure("Erreur lors de la création de l'utilisateur");
    }
    catch (SqlException ex) when (ex.Number == -2) // Timeout
    {
        logger.LogError(ex, "Database timeout while creating user: {Username}", request.Username);
        return Result.Failure("La base de données met trop de temps à répondre");
    }
    catch (Exception ex)
    {
        logger.LogError(ex, "Unexpected error while creating user: {Username}", request.Username);
        return Result.Failure("Une erreur inattendue s'est produite");
    }
}

private static bool IsUniqueConstraintViolation(DbUpdateException ex)
{
    return ex.InnerException is SqlException sqlEx && 
           (sqlEx.Number == 2627 || sqlEx.Number == 2601);
}

{
    if (!optionsBuilder.IsConfigured)
    {
        optionsBuilder
            .UseSqlServer(connectionString)
            .LogTo(
                message => logger.LogDebug(message),
                new[] { DbLoggerCategory.Database.Command.Name },
                LogLevel.Information,
                DbContextLoggerOptions.SingleLine | DbContextLoggerOptions.UtcTime
            );
    }
}

Stratégie de Validation

  3. Business Logic → Règles métier complexes
  4. Database → Contraintes, triggers
Validation Robuste avec FluentValidation
// DTO avec validation
public class CreateUserRequest
{
    public string Username { get; set; }
    public string Email { get; set; }
    public string Password { get; set; }
    public DateTime? BirthDate { get; set; }
}

// Validateur FluentValidation
public class CreateUserRequestValidator : AbstractValidator<CreateUserRequest>
{
    private readonly IUserRepository userRepository;

    public CreateUserRequestValidator(IUserRepository userRepository)
    {
        this.userRepository = userRepository;

        RuleFor(x => x.Username)
            .NotEmpty().WithMessage("Le nom d'utilisateur est obligatoire")
            .Length(3, 50).WithMessage("Le nom d'utilisateur doit contenir entre 3 et 50 caractères")
            .Matches("^[a-zA-Z0-9_]+$").WithMessage("Le nom d'utilisateur ne peut contenir que des lettres, chiffres et _")
            .MustAsync(BeUniqueUsername).WithMessage("Ce nom d'utilisateur existe déjà");

        RuleFor(x => x.Email)
            .NotEmpty().WithMessage("L'email est obligatoire")
            .EmailAddress().WithMessage("Format d'email invalide")
            .MustAsync(BeUniqueEmail).WithMessage("Cet email est déjà utilisé");

        RuleFor(x => x.Password)
            .NotEmpty().WithMessage("Le mot de passe est obligatoire")
            .MinimumLength(8).WithMessage("Le mot de passe doit contenir au moins 8 caractères")
            .Matches(@"^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]")
            .WithMessage("Le mot de passe doit contenir au moins une minuscule, une majuscule, un chiffre et un caractère spécial");

        RuleFor(x => x.BirthDate)
            .LessThan(DateTime.Today.AddYears(-13))
            .WithMessage("L'utilisateur doit avoir au moins 13 ans")
            .When(x => x.BirthDate.HasValue);
    }

    private async Task<bool> BeUniqueUsername(string username, CancellationToken cancellationToken)
    {
        return !await userRepository.ExistsAsync(u => u.Username == username);
    }

    private async Task<bool> BeUniqueEmail(string email, CancellationToken cancellationToken)
    {
        return !await userRepository.ExistsAsync(u => u.Email == email);
    }
}

Sécurisation des Mots de Passe

Hashage Sécurisé avec BCrypt
public class PasswordService
{
    public string HashPassword(string password)
    {
        // Générer un salt aléatoire et hasher le mot de passe
        return BCrypt.Net.BCrypt.HashPassword(password, BCrypt.Net.BCrypt.GenerateSalt(12));
    }

    public bool VerifyPassword(string password, string hash)
    {
        return BCrypt.Net.BCrypt.Verify(password, hash);
    }
}

// Utilisation lors de la création d'utilisateur
public async Task<User> CreateUserAsync(CreateUserRequest request)
{
    var user = new User
    {
        Username = request.Username,
        Email = request.Email,
        PasswordHash = passwordService.HashPassword(request.Password), // ✅ Jamais en clair
        CreatedAt = DateTime.UtcNow
    };

    context.Users.Add(user);
    await context.SaveChangesAsync();

    // ⚠️ Ne pas retourner le hash du mot de passe
    user.PasswordHash = null;
    return user;
}

Masquage et Chiffrement
public class User
{
    public int Id { get; set; }
    public string Username { get; set; }
    public string Email { get; set; }
    
    [JsonIgnore] // Ne pas sérialiser
    public string PasswordHash { get; set; }
    
    // Propriété chiffrée pour données sensibles
    [Column("EncryptedSSN")]
    public string SocialSecurityNumber { get; set; }
    
    public DateTime CreatedAt { get; set; }
}

// Service de chiffrement pour données sensibles
public class EncryptionService
{
    private readonly IDataProtector protector;

    public EncryptionService(IDataProtectionProvider provider)
    {
        protector = provider.CreateProtector("UserData.SSN");
    }

    public string Encrypt(string plainText)
    {
        return string.IsNullOrEmpty(plainText) ? null : protector.Protect(plainText);
    }

    public string Decrypt(string cipherText)
    {
        return string.IsNullOrEmpty(cipherText) ? null : protector.Unprotect(cipherText);
    }
}

Architecture et Bonnes Pratiques MOYEN

Pattern Repository et Unit of Work

Repository Pattern
// Interface générique
public interface IRepository<T> where T : class
{
    Task<T> GetByIdAsync(int id);
    Task<IEnumerable<T>> GetAllAsync();
    Task<IEnumerable<T>> FindAsync(Expression<Func<T, bool>> predicate);
    Task AddAsync(T entity);
    void Update(T entity);
    void Remove(T entity);
}

// Implémentation générique
public class Repository<T> : IRepository<T> where T : class
{
    protected readonly DbContext context;
    protected readonly DbSet<T> dbSet;

    public Repository(DbContext context)
    {
        this.context = context;
        this.dbSet = context.Set<T>();
    }

    public virtual async Task<T> GetByIdAsync(int id)
    {
        return await dbSet.FindAsync(id);
    }

    public virtual async Task<IEnumerable<T>> GetAllAsync()
    {
        return await dbSet.AsNoTracking().ToListAsync();
    }

    public virtual async Task<IEnumerable<T>> FindAsync(Expression<Func<T, bool>> predicate)
    {
        return await dbSet.AsNoTracking().Where(predicate).ToListAsync();
    }

    public virtual async Task AddAsync(T entity)
    {
        await dbSet.AddAsync(entity);
    }

    public virtual void Update(T entity)
    {
        dbSet.Update(entity);
    }

    public virtual void Remove(T entity)
    {
        dbSet.Remove(entity);
    }
}

// Repository spécialisé
public interface IUserRepository : IRepository<User>
{
    Task<User> GetByUsernameAsync(string username);
    Task<bool> ExistsAsync(Expression<Func<User, bool>> predicate);
}

public class UserRepository : Repository<User>, IUserRepository
{
    public UserRepository(AppDbContext context) : base(context) { }

    public async Task<User> GetByUsernameAsync(string username)
    {
        return await dbSet.AsNoTracking()
            .FirstOrDefaultAsync(u => u.Username == username);
    }

    public async Task<bool> ExistsAsync(Expression<Func<User, bool>> predicate)
    {
        return await dbSet.AsNoTracking().AnyAsync(predicate);
    }
}

Unit of Work Pattern

Gestion Transactionnelle
public interface IUnitOfWork : IDisposable
{
    IUserRepository Users { get; }
    IProductRepository Products { get; }
    IOrderRepository Orders { get; }
    
    Task<int> SaveChangesAsync();
    Task BeginTransactionAsync();
    Task CommitTransactionAsync();
    Task RollbackTransactionAsync();
}

public class UnitOfWork : IUnitOfWork
{
    private readonly AppDbContext context;
    private IDbContextTransaction transaction;

    public UnitOfWork(AppDbContext context)
    {
        this.context = context;
        Users = new UserRepository(context);
        Products = new ProductRepository(context);
        Orders = new OrderRepository(context);
    }

    public IUserRepository Users { get; }
    public IProductRepository Products { get; }
    public IOrderRepository Orders { get; }

    public async Task<int> SaveChangesAsync()
    {
        return await context.SaveChangesAsync();
    }

    public async Task BeginTransactionAsync()
    {
        transaction = await context.Database.BeginTransactionAsync();
    }

    public async Task CommitTransactionAsync()
    {
        try
        {
            await SaveChangesAsync();
            await transaction.CommitAsync();
        }
        catch
        {
            await RollbackTransactionAsync();
            throw;
        }
        finally
        {
            transaction?.Dispose();
            transaction = null;
        }
    }

    public async Task RollbackTransactionAsync()
    {
        if (transaction != null)
        {
            await transaction.RollbackAsync();
            transaction.Dispose();
            transaction = null;
        }
    }

    public void Dispose()
    {
        transaction?.Dispose();
        context?.Dispose();
    }
}

// ❌ CODE VULNÉRABLE - Trouvez les problèmes !
public class UserController : ControllerBase
{
    [HttpPost("login")]
    public async Task<IActionResult> Login(string username, string password)
    {
        var user = await context.Users
            .FromSqlRaw($"SELECT * FROM Users WHERE Username = '{username}' AND Password = '{password}'")
            .FirstOrDefaultAsync();
            
        if (user != null)
        {
            logger.LogInformation($"User {username} logged in with password {password}");
            return Ok(user);
        }
        
        return Unauthorized("Invalid credentials");
    }
}
