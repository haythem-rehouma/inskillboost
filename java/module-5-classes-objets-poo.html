Module 5 - Classes et Objets POO

Fondamentaux de la Programmation Orientée Objet

Introduction à la Programmation Orientée Objet

La Programmation Orientée Objet (POO) est un paradigme de programmation qui organise le code autour d'objets plutôt que de fonctions et de logique.

Encapsulation

Regrouper les données (attributs) et les comportements (méthodes) dans une même entité (classe).

Héritage

Capacité d'une classe à hériter des propriétés et méthodes d'une autre classe.

Polymorphisme

Capacité d'un objet à prendre plusieurs formes selon le contexte.

Abstraction

Masquer la complexité interne et ne montrer que l'interface nécessaire.

Définition de Classes

Une classe est un modèle ou un plan pour créer des objets. Elle définit les attributs (données) et les méthodes (comportements) que les objets de cette classe auront.

Syntaxe de Base d'une Classe

public class Personne {
    // Attributs (variables d'instance)
    private String nom;
    private String prenom;
    private int age;
    private String email;
    
    // Constructeur
    public Personne(String nom, String prenom, int age, String email) {
        this.nom = nom;
        this.prenom = prenom;
        this.age = age;
        this.email = email;
    }
    
    // Méthodes (comportements)
    public void sePresenter() {
        System.out.println("Bonjour, je suis " + prenom + " " + nom);
        System.out.println("J'ai " + age + " ans");
    }
    
    public boolean estMajeur() {
        return age >= 18;
    }
    
    // Getters et Setters (Encapsulation)
    public String getNom() {
        return nom;
    }
    
    public void setNom(String nom) {
        this.nom = nom;
    }
    
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        if (age >= 0 && age <= 120) {
            this.age = age;
        } else {
            System.out.println("Age invalide !");
        }
    }
}
Éléments Clés d'une Classe :
  • Modificateurs d'accès : public, private, protected
  • Attributs : Variables qui stockent l'état de l'objet
  • Constructeurs : Méthodes spéciales pour initialiser les objets
  • Méthodes : Fonctions qui définissent les comportements
  • this : Référence à l'objet courant

Constructeurs

Les constructeurs sont des méthodes spéciales utilisées pour initialiser les objets lors de leur création.

Types de Constructeurs

public class Voiture {
    private String marque;
    private String modele;
    private int annee;
    private double prix;
    
    // 1. Constructeur par défaut (sans paramètres)
    public Voiture() {
        this.marque = "Inconnue";
        this.modele = "Inconnu";
        this.annee = 2024;
        this.prix = 0.0;
    }
    
    // 2. Constructeur avec paramètres
    public Voiture(String marque, String modele, int annee, double prix) {
        this.marque = marque;
        this.modele = modele;
        this.annee = annee;
        this.prix = prix;
    }
    
    // 3. Constructeur avec validation
    public Voiture(String marque, String modele, int annee) {
        this(marque, modele, annee, 0.0); // Appel du constructeur principal
        
        // Validation de l'année
        if (annee < 1886 || annee > 2024) {
            System.out.println("Année invalide, définition par défaut à 2024");
            this.annee = 2024;
        }
    }
    
    // Méthode pour afficher les informations
    public void afficherInfos() {
        System.out.println("Voiture: " + marque + " " + modele + 
                          " (" + annee + ") - " + prix + "€");
    }
}

Encapsulation et Modificateurs d'Accès

L'encapsulation consiste à cacher les détails internes d'une classe et à contrôler l'accès aux données via des méthodes publiques.

Modificateurs d'Accès

  • private : Accessible uniquement dans la classe
  • protected : Accessible dans le package et les sous-classes
  • public : Accessible partout
  • (default) : Accessible dans le package
Bonnes Pratiques
  • Attributs private
  • Méthodes public pour l'interface
  • Getters/Setters pour l'accès contrôlé
  • Validation dans les setters

Exemple Complet avec Encapsulation

public class CompteBancaire {
    // Attributs privés (encapsulation)
    private String numeroCompte;
    private String titulaire;
    private double solde;
    private static int compteurComptes = 0;
    
    // Constructeur
    public CompteBancaire(String titulaire, double soldeInitial) {
        this.numeroCompte = "CPT" + String.format("%06d", ++compteurComptes);
        this.titulaire = titulaire;
        this.solde = soldeInitial >= 0 ? soldeInitial : 0;
    }
    
    // Méthodes métier
    public boolean deposer(double montant) {
        if (montant > 0) {
            solde += montant;
            System.out.println("Dépôt de " + montant + "€ effectué");
            return true;
        }
        System.out.println("Montant invalide");
        return false;
    }
    
    public boolean retirer(double montant) {
        if (montant > 0 && montant <= solde) {
            solde -= montant;
            System.out.println("Retrait de " + montant + "€ effectué");
            return true;
        }
        System.out.println("Retrait impossible");
        return false;
    }
    
    // Getters (lecture seule pour certains attributs)
    public String getNumeroCompte() {
        return numeroCompte;
    }
    
    public String getTitulaire() {
        return titulaire;
    }
    
    public double getSolde() {
        return solde;
    }
    
    // Setter avec validation
    public void setTitulaire(String titulaire) {
        if (titulaire != null && !titulaire.trim().isEmpty()) {
            this.titulaire = titulaire;
        } else {
            System.out.println("Nom du titulaire invalide");
        }
    }
    
    // Méthode toString pour affichage
    @Override
    public String toString() {
        return "Compte " + numeroCompte + " - " + titulaire + " : " + solde + "€";
    }
}

Exercice 1 : Classe Étudiant

Objectif : Créer une classe Étudiant complète avec encapsulation.

Spécifications :

  • Attributs privés : nom, prenom, numeroEtudiant, notes (tableau)
  • Constructeur avec validation
  • Méthodes : ajouterNote(), calculerMoyenne(), afficherBulletin()
  • Getters/Setters appropriés
import java.util.ArrayList;
import java.util.List;

public class Etudiant {
    private String nom;
    private String prenom;
    private String numeroEtudiant;
    private List notes;
    private static int compteurEtudiants = 0;
    
    public Etudiant(String nom, String prenom) {
        this.nom = nom;
        this.prenom = prenom;
        this.numeroEtudiant = "ETU" + String.format("%05d", ++compteurEtudiants);
        this.notes = new ArrayList<>();
    }
    
    public void ajouterNote(double note) {
        if (note >= 0 && note <= 20) {
            notes.add(note);
            System.out.println("Note " + note + " ajoutée pour " + prenom + " " + nom);
        } else {
            System.out.println("Note invalide (doit être entre 0 et 20)");
        }
    }
    
    public double calculerMoyenne() {
        if (notes.isEmpty()) {
            return 0;
        }
        
        double somme = 0;
        for (double note : notes) {
            somme += note;
        }
        return somme / notes.size();
    }
    
    public void afficherBulletin() {
        System.out.println("=== BULLETIN DE " + prenom.toUpperCase() + " " + nom.toUpperCase() + " ===");
        System.out.println("Numéro étudiant : " + numeroEtudiant);
        System.out.println("Notes : " + notes);
        System.out.println("Moyenne : " + String.format("%.2f", calculerMoyenne()) + "/20");
        System.out.println("Mention : " + obtenirMention());
        System.out.println("=====================================");
    }
    
    private String obtenirMention() {
        double moyenne = calculerMoyenne();
        if (moyenne >= 16) return "Très Bien";
        if (moyenne >= 14) return "Bien";
        if (moyenne >= 12) return "Assez Bien";
        if (moyenne >= 10) return "Passable";
        return "Insuffisant";
    }
    
    // Getters
    public String getNom() { return nom; }
    public String getPrenom() { return prenom; }
    public String getNumeroEtudiant() { return numeroEtudiant; }
    public List getNotes() { return new ArrayList<>(notes); }
}

Exercice 2 : Système de Gestion de Véhicules

Objectif : Créer un système de gestion pour un garage.

Classes à créer :

  • Vehicule : Classe de base avec marque, modèle, année, kilométrage
  • Garage : Classe pour gérer une collection de véhicules
  • TestGarage : Classe principale pour tester le système
Classe Vehicule :
public class Vehicule {
    private String marque;
    private String modele;
    private int annee;
    private double kilometrage;
    private boolean disponible;
    
    public Vehicule(String marque, String modele, int annee) {
        this.marque = marque;
        this.modele = modele;
        this.annee = annee;
        this.kilometrage = 0.0;
        this.disponible = true;
    }
    
    public void rouler(double km) {
        if (km > 0) {
            this.kilometrage += km;
            System.out.println(marque + " " + modele + " a roulé " + km + " km");
        }
    }
    
    public void louer() {
        if (disponible) {
            disponible = false;
            System.out.println(marque + " " + modele + " loué avec succès");
        } else {
            System.out.println("Véhicule non disponible");
        }
    }
    
    public void retourner() {
        disponible = true;
        System.out.println(marque + " " + modele + " retourné");
    }
    
    public void afficherInfos() {
        String statut = disponible ? "Disponible" : "Loué";
        System.out.println(marque + " " + modele + " (" + annee + ") - " + 
                          kilometrage + " km - " + statut);
    }
    
    // Getters et Setters
    public String getMarque() { return marque; }
    public String getModele() { return modele; }
    public int getAnnee() { return annee; }
    public double getKilometrage() { return kilometrage; }
    public boolean isDisponible() { return disponible; }
}

Architecture du Projet :

Ce projet démontre l'utilisation de classes multiples, l'encapsulation, et la gestion d'état.

Structure des Fichiers :
  • Livre.java - Représente un livre
