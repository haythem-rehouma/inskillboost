Module 5 - Classes et Objets POO

Fondamentaux de la Programmation Orientée Objet

Qu'est-ce que la Programmation Orientée Objet ?

La Programmation Orientée Objet (POO) est une façon de programmer qui imite le monde réel. Au lieu d'écrire une suite d'instructions, on crée des "objets" qui représentent des choses concrètes et qui peuvent interagir entre eux.

Analogie : Programmation Traditionnelle vs POO

Programmation Traditionnelle (Procédurale) :

// Approche procédurale : tout dans des fonctions séparées
String nomVoiture = "Toyota Camry";
String couleurVoiture = "Rouge";
boolean voitureDemarree = false;

void demarrerVoiture() {
    voitureDemarree = true;
    System.out.println(nomVoiture + " démarrée");
}

void arreterVoiture() {
    voitureDemarree = false;
    System.out.println(nomVoiture + " arrêtée");
}

// Problème : données et comportements séparés
// Difficile à maintenir si on a plusieurs voitures !

Programmation Orientée Objet :

// Approche POO : tout regroupé dans une classe
public class Voiture {
    // DONNÉES (attributs)
    private String nom;
    private String couleur;
    private boolean demarree;
    
    // COMPORTEMENTS (méthodes)
    public void demarrer() {
        demarree = true;
        System.out.println(nom + " démarrée");
    }
    
    public void arreter() {
        demarree = false;
        System.out.println(nom + " arrêtée");
    }
}

// Avantage : données et comportements unis
// Facile de créer plusieurs voitures !

Les 4 Piliers de la POO Expliqués Simplement

1. Encapsulation

Définition : Regrouper les données et les comportements dans une même "boîte" (classe)

Analogie : Une télécommande TV

  • Circuits internes cachés (attributs privés)
  • Boutons visibles (méthodes publiques)
  • On utilise les boutons sans connaître l'électronique
2. Héritage

Définition : Une classe peut hériter des propriétés d'une autre classe

Analogie : Famille génétique

  • Enfant hérite des traits des parents
  • Peut avoir ses propres traits uniques
  • Évite de redéfinir ce qui existe déjà
3. Polymorphisme

Définition : Un objet peut prendre plusieurs formes selon le contexte

Analogie : Une personne multi-rôles

  • Même personne = mère, employée, conductrice
  • Comportement différent selon le contexte
  • Interface commune, implémentation différente
4. Abstraction

Définition : Masquer la complexité et ne montrer que l'essentiel

Analogie : Conduire une voiture

  • Volant, pédales, levier = interface simple
  • Moteur, transmission = complexité cachée
  • On conduit sans connaître la mécanique

Exemple Concret : Modéliser une Personne

/**
 * AVANT la POO : Comment on pourrait mal s'y prendre
 */
public class SansPoO {
    public static void main(String[] args) {
        // Variables séparées pour chaque personne
        String nomPersonne1 = "Alice";
        int agePersonne1 = 25;
        String emailPersonne1 = "alice@email.com";
        
        String nomPersonne2 = "Bob";
        int agePersonne2 = 30;
        String emailPersonne2 = "bob@email.com";
        
        // Fonctions séparées (pas liées aux données)
        sePresenter(nomPersonne1, agePersonne1);
        sePresenter(nomPersonne2, agePersonne2);
        
        // Problèmes :
        // 1. Données éparpillées
        // 2. Pas de lien logique entre nom, âge, email d'une même personne
        // 3. Fonctions séparées des données
        // 4. Difficile à maintenir et étendre
    }
    
    public static void sePresenter(String nom, int age) {
        System.out.println("Je suis " + nom + " et j'ai " + age + " ans");
    }
}

/**
 * AVEC la POO : Approche élégante et organisée
 */
public class AvecPoO {
    public static void main(String[] args) {
        // Création d'objets qui représentent des personnes réelles
        Personne alice = new Personne("Alice", 25, "alice@email.com");
        Personne bob = new Personne("Bob", 30, "bob@email.com");
        
        // Chaque objet a ses propres données ET comportements
        alice.sePresenter(); // Alice se présente elle-même
        bob.sePresenter();   // Bob se présente lui-même
        
        // Avantages :
        // 1. Données et comportements unis
        // 2. Chaque personne est une entité complète
        // 3. Code réutilisable et extensible
        // 4. Modélisation naturelle du monde réel
    }
}

// La classe Personne encapsule TOUT ce qui concerne une personne
class Personne {
    // DONNÉES (attributs) - ce qu'une personne "a"
    private String nom;
    private int age;
    private String email;
    
    // CONSTRUCTEUR - comment créer une personne
    public Personne(String nom, int age, String email) {
        this.nom = nom;
        this.age = age;
        this.email = email;
    }
    
    // COMPORTEMENTS (méthodes) - ce qu'une personne "fait"
    public void sePresenter() {
        System.out.println("Bonjour, je suis " + nom + " et j'ai " + age + " ans");
    }
    
    public void vieillir() {
        age++;
        System.out.println(nom + " a maintenant " + age + " ans");
    }
    
    public boolean estMajeur() {
        return age >= 18;
    }
}

/*
RÉSULTAT :
Bonjour, je suis Alice et j'ai 25 ans
Bonjour, je suis Bob et j'ai 30 ans

AVANTAGES POO :
✅ Code organisé et logique
✅ Réutilisabilité (créer autant de personnes qu'on veut)
✅ Maintenance facile (tout est au même endroit)
✅ Modélisation naturelle du monde réel
*/

Qu'est-ce qu'une Classe ? Qu'est-ce qu'un Objet ?

Une classe est comme un plan architectural ou un moule à gâteau. Elle décrit comment quelque chose doit être construit, mais ce n'est pas la chose elle-même. Un objet est une instance concrète créée à partir de ce plan.

Analogie : Plan de Maison vs Maisons Réelles
🏗️ Classe = Plan Architectural
  • Décrit COMMENT construire une maison
  • Spécifie : 3 chambres, 2 salles de bain, cuisine
  • Définit les comportements : ouvrir porte, allumer lumière
  • UN SEUL plan peut servir pour plusieurs maisons
🏠 Objet = Maison Réelle
  • Maison CONCRÈTE construite selon le plan
  • A une adresse spécifique (123 rue Java)
  • Couleurs et meubles personnalisés
  • PLUSIEURS maisons peuvent être construites avec le même plan

Exemple Concret : Classe Voiture

/**
 * CLASSE Voiture = Plan pour créer des voitures
 */
public class Voiture {
    // ATTRIBUTS (données) - Ce qu'une voiture "a"
    private String marque;      // Toyota, Renault, etc.
    private String modele;      // Camry, Clio, etc.
    private String couleur;     // Rouge, Bleu, etc.
    private int annee;          // 2020, 2021, etc.
    private boolean demarree;   // true/false
    private double kilometrage; // Nombre de km parcourus
    
    // CONSTRUCTEUR - Comment créer une voiture
    public Voiture(String marque, String modele, String couleur, int annee) {
        this.marque = marque;
        this.modele = modele;
        this.couleur = couleur;
        this.annee = annee;
        this.demarree = false;    // Voiture arrêtée par défaut
        this.kilometrage = 0.0;   // Neuve par défaut
        
        System.out.println("🚗 Nouvelle voiture créée: " + marque + " " + modele + " " + couleur);
    }
    
    // MÉTHODES (comportements) - Ce qu'une voiture "fait"
    public void demarrer() {
        if (!demarree) {
            demarree = true;
            System.out.println("🔥 " + marque + " " + modele + " démarre !");
        } else {
            System.out.println("⚠️  La voiture est déjà démarrée");
        }
    }
    
    public void arreter() {
        if (demarree) {
            demarree = false;
            System.out.println("🛑 " + marque + " " + modele + " s'arrête");
        } else {
            System.out.println("⚠️  La voiture est déjà arrêtée");
        }
    }
    
    public void rouler(double distance) {
        if (demarree && distance > 0) {
            kilometrage += distance;
            System.out.printf("🛣️  %s %s roule %.1f km (Total: %.1f km)%n", 
                            marque, modele, distance, kilometrage);
        } else if (!demarree) {
            System.out.println("❌ Impossible de rouler : voiture arrêtée");
        } else {
            System.out.println("❌ Distance invalide");
        }
    }
    
    public void afficherInfos() {
        System.out.println("=== INFORMATIONS VOITURE ===");
        System.out.println("Marque: " + marque);
        System.out.println("Modèle: " + modele);
        System.out.println("Couleur: " + couleur);
        System.out.println("Année: " + annee);
        System.out.println("Kilométrage: " + kilometrage + " km");
        System.out.println("État: " + (demarree ? "Démarrée" : "Arrêtée"));
    }
    
    // GETTERS - Pour lire les attributs privés
    public String getMarque() { return marque; }
    public String getModele() { return modele; }
    public String getCouleur() { return couleur; }
    public int getAnnee() { return annee; }
    public boolean estDemarree() { return demarree; }
    public double getKilometrage() { return kilometrage; }
}

/**
 * UTILISATION de la classe Voiture
 */
public class TestVoiture {
    public static void main(String[] args) {
        System.out.println("=== CRÉATION D'OBJETS VOITURE ===");
        
        // Créer plusieurs objets (instances) de la classe Voiture
        Voiture voiture1 = new Voiture("Toyota", "Camry", "Rouge", 2022);
        Voiture voiture2 = new Voiture("Renault", "Clio", "Bleue", 2021);
        Voiture voiture3 = new Voiture("BMW", "X3", "Noire", 2023);
        
        System.out.println("\n=== TEST DES COMPORTEMENTS ===");
        
        // Chaque objet a ses propres données et peut faire ses propres actions
        voiture1.demarrer();
        voiture1.rouler(50.5);
        voiture1.afficherInfos();
        
        System.out.println();
        
        voiture2.demarrer();
        voiture2.rouler(25.0);
        voiture2.arreter();
        voiture2.rouler(10.0); // Tentative de rouler arrêtée
        voiture2.afficherInfos();

        // voiture3 reste arrêtée
        voiture3.afficherInfos();
        
        System.out.println("\n💡 OBSERVATIONS :");
        System.out.println("- Chaque voiture a ses propres valeurs d'attributs");
        System.out.println("- Les actions sur voiture1 n'affectent pas voiture2");
        System.out.println("- Même classe, mais objets indépendants !");
    }
}

Relation Classe ↔ Objet :
  • 1 classe → peut créer plusieurs objets
  • Classe : modèle, template, blueprint
  • Objet : instance concrète, entité réelle
  • Chaque objet a ses propres valeurs d'attributs
  • Tous les objets partagent les mêmes méthodes

Constructeurs

Les constructeurs sont des méthodes spéciales utilisées pour initialiser les objets lors de leur création.

Types de Constructeurs

public class Voiture {
    private String marque;
    private String modele;
    private int annee;
    private double prix;
    
    // 1. Constructeur par défaut (sans paramètres)
    public Voiture() {
        this.marque = "Inconnue";
        this.modele = "Inconnu";
        this.annee = 2024;
        this.prix = 0.0;
    }
    
    // 2. Constructeur avec paramètres
    public Voiture(String marque, String modele, int annee, double prix) {
        this.marque = marque;
        this.modele = modele;
        this.annee = annee;
        this.prix = prix;
    }
    
    // 3. Constructeur avec validation
    public Voiture(String marque, String modele, int annee) {
        this(marque, modele, annee, 0.0); // Appel du constructeur principal
        
        // Validation de l'année
        if (annee < 1886 || annee > 2024) {
            System.out.println("Année invalide, définition par défaut à 2024");
            this.annee = 2024;
        }
    }
    
    // Méthode pour afficher les informations
    public void afficherInfos() {
        System.out.println("Voiture: " + marque + " " + modele + 
                          " (" + annee + ") - " + prix + "€");
    }
}

Encapsulation et Modificateurs d'Accès

L'encapsulation consiste à cacher les détails internes d'une classe et à contrôler l'accès aux données via des méthodes publiques.

Modificateurs d'Accès

  • private : Accessible uniquement dans la classe
  • protected : Accessible dans le package et les sous-classes
  • public : Accessible partout
  • (default) : Accessible dans le package
Bonnes Pratiques
  • Attributs private
  • Méthodes public pour l'interface
  • Getters/Setters pour l'accès contrôlé
  • Validation dans les setters

Exemple Complet avec Encapsulation

public class CompteBancaire {
    // Attributs privés (encapsulation)
    private String numeroCompte;
    private String titulaire;
    private double solde;
    private static int compteurComptes = 0;
    
    // Constructeur
    public CompteBancaire(String titulaire, double soldeInitial) {
        this.numeroCompte = "CPT" + String.format("%06d", ++compteurComptes);
        this.titulaire = titulaire;
        this.solde = soldeInitial >= 0 ? soldeInitial : 0;
    }
    
    // Méthodes métier
    public boolean deposer(double montant) {
        if (montant > 0) {
            solde += montant;
            System.out.println("Dépôt de " + montant + "€ effectué");
            return true;
        }
        System.out.println("Montant invalide");
        return false;
    }
    
    public boolean retirer(double montant) {
        if (montant > 0 && montant <= solde) {
            solde -= montant;
            System.out.println("Retrait de " + montant + "€ effectué");
            return true;
        }
        System.out.println("Retrait impossible");
        return false;
    }
    
    // Getters (lecture seule pour certains attributs)
    public String getNumeroCompte() {
        return numeroCompte;
    }
    
    public String getTitulaire() {
        return titulaire;
    }
    
    public double getSolde() {
        return solde;
    }
    
    // Setter avec validation
    public void setTitulaire(String titulaire) {
        if (titulaire != null && !titulaire.trim().isEmpty()) {
            this.titulaire = titulaire;
        } else {
            System.out.println("Nom du titulaire invalide");
        }
    }
    
    // Méthode toString pour affichage
    @Override
    public String toString() {
        return "Compte " + numeroCompte + " - " + titulaire + " : " + solde + "€";
    }
}

Exercice 1 : Classe Étudiant

Objectif : Créer une classe Étudiant complète avec encapsulation.

Spécifications :

  • Attributs privés : nom, prenom, numeroEtudiant, notes (tableau)
  • Constructeur avec validation
  • Méthodes : ajouterNote(), calculerMoyenne(), afficherBulletin()
  • Getters/Setters appropriés
import java.util.ArrayList;
import java.util.List;

public class Etudiant {
    private String nom;
    private String prenom;
    private String numeroEtudiant;
    private List notes;
    private static int compteurEtudiants = 0;
    
    public Etudiant(String nom, String prenom) {
        this.nom = nom;
        this.prenom = prenom;
        this.numeroEtudiant = "ETU" + String.format("%05d", ++compteurEtudiants);
        this.notes = new ArrayList<>();
    }
    
    public void ajouterNote(double note) {
        if (note >= 0 && note <= 20) {
            notes.add(note);
            System.out.println("Note " + note + " ajoutée pour " + prenom + " " + nom);
        } else {
            System.out.println("Note invalide (doit être entre 0 et 20)");
        }
    }
    
    public double calculerMoyenne() {
        if (notes.isEmpty()) {
            return 0;
        }
        
        double somme = 0;
        for (double note : notes) {
            somme += note;
        }
        return somme / notes.size();
    }
    
    public void afficherBulletin() {
        System.out.println("=== BULLETIN DE " + prenom.toUpperCase() + " " + nom.toUpperCase() + " ===");
        System.out.println("Numéro étudiant : " + numeroEtudiant);
        System.out.println("Notes : " + notes);
        System.out.println("Moyenne : " + String.format("%.2f", calculerMoyenne()) + "/20");
        System.out.println("Mention : " + obtenirMention());
        System.out.println("=====================================");
    }
    
    private String obtenirMention() {
        double moyenne = calculerMoyenne();
        if (moyenne >= 16) return "Très Bien";
        if (moyenne >= 14) return "Bien";
        if (moyenne >= 12) return "Assez Bien";
        if (moyenne >= 10) return "Passable";
        return "Insuffisant";
    }
    
    // Getters
    public String getNom() { return nom; }
    public String getPrenom() { return prenom; }
    public String getNumeroEtudiant() { return numeroEtudiant; }
    public List getNotes() { return new ArrayList<>(notes); }
}

Exercice 2 : Système de Gestion de Véhicules

Objectif : Créer un système de gestion pour un garage.

Classes à créer :

  • Vehicule : Classe de base avec marque, modèle, année, kilométrage
  • Garage : Classe pour gérer une collection de véhicules
  • TestGarage : Classe principale pour tester le système
Classe Vehicule :
public class Vehicule {
    private String marque;
    private String modele;
    private int annee;
    private double kilometrage;
    private boolean disponible;
    
    public Vehicule(String marque, String modele, int annee) {
        this.marque = marque;
        this.modele = modele;
        this.annee = annee;
        this.kilometrage = 0.0;
        this.disponible = true;
    }
    
    public void rouler(double km) {
        if (km > 0) {
            this.kilometrage += km;
            System.out.println(marque + " " + modele + " a roulé " + km + " km");
        }
    }
    
    public void louer() {
        if (disponible) {
            disponible = false;
            System.out.println(marque + " " + modele + " loué avec succès");
        } else {
            System.out.println("Véhicule non disponible");
        }
    }
    
    public void retourner() {
        disponible = true;
        System.out.println(marque + " " + modele + " retourné");
    }
    
    public void afficherInfos() {
        String statut = disponible ? "Disponible" : "Loué";
        System.out.println(marque + " " + modele + " (" + annee + ") - " + 
                          kilometrage + " km - " + statut);
    }
    
    // Getters et Setters
    public String getMarque() { return marque; }
    public String getModele() { return modele; }
    public int getAnnee() { return annee; }
    public double getKilometrage() { return kilometrage; }
    public boolean isDisponible() { return disponible; }
}
