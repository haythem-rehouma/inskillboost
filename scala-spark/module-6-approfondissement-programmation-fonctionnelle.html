Questions et Réponses

Qu'est-ce que le symbole _ ?

En Scala, le symbole _ est utilisé comme un paramètre anonyme dans les fonctions anonymes (ou lambdas). Il représente un ou plusieurs arguments passés à  une fonction.

Exemple - Création et transformation de liste 
// 1. Création de la liste
val list = List(1, 2, 3, 4, 5)
// Résultat: List(1, 2, 3, 4, 5)

// 2. Application du filtre
list.filter(_ % 2 == 0)
// Résultat: List(2, 4)

// 3. Application de la transformation
.map(_ * 2)
// Résultat: List(4, 8)
Explication détaillée :
  • list.filter(_ % 2 == 0) : Ici, _ représente chaque élément de la liste. Chaque élément est testé pour voir s'il est divisible par 2.
  • .map(_ * 2) : De même, _ représente chaque élément de la liste filtrée, et cet élément est multiplié par 2.

L'utilisation de _ simplifie l'écriture de fonctions anonymes en évitant de devoir nommer explicitement les paramètres.

Support automatique du pattern matching

La particularité principale des objets case en Scala est qu'ils supportent automatiquement le pattern matching.

Qu'est-ce que le pattern matching ?

Le pattern matching permet de vérifier une valeur contre un modèle et d'exécuter du code en fonction du modèle qui correspond. C'est une version avancée de switch, mais beaucoup plus puissante.

Exemple avec case class et pattern matching 
sealed trait Animal
case class Dog(name: String) extends Animal
case class Cat(name: String) extends Animal

def greet(animal: Animal): String = animal match {
  case Dog(name) => s"Hello, $name the dog!"
  case Cat(name) => s"Hello, $name the cat!"
}

val myDog = Dog("Buddy")
println(greet(myDog)) // Output: Hello, Buddy the dog!
Autres fonctionnalités automatiques :
  • Méthode apply automatique : Pas besoin d'appeler new
  • Méthode copy automatique : Crée une copie en modifiant certaines valeurs
  • égalité structurelle : Deux instances avec les mêmes valeurs sont égales
  • Sérialisation facile : Les case class sont facilement sérialisables

Qu'est-ce qu'une méthode générique ?

Une méthode générique peut fonctionner avec différents types de données, sans être limitée à  un type spécifique. Cela permet d'écrire du code plus flexible et réutilisable.

Exemple - Méthode générique 
// Sans génériques (répétitif)
def getFirstInt(list: List[Int]): Int = list.head
def getFirstString(list: List[String]): String = list.head

// Avec génériques (flexible)
def getFirst[T](list: List[T]): T = list.head

// Utilisation
val intList = List(1, 2, 3)
val stringList = List("a", "b", "c")

println(getFirst(intList))    // Sortie : 1
println(getFirst(stringList)) // Sortie : "a"
Avantages des méthodes génériques :
  • Réutilisabilité : Une seule méthode pour tous les types
  • Sécurité de type : Le compilateur vérifie la cohérence des types
  • Moins de code : évite la duplication de code

Un trait peut contenir des méthodes concrètes et abstraites.

C'est quoi un trait en Scala ?

Pense à  un trait comme à  une sorte de plan ou de recette que tu peux appliquer à  plusieurs objets ou classes. Il te permet de dire : "Voici les choses que cet objet ou cette classe doit pouvoir faire."

Exemple - Définition et utilisation d'un trait 
trait Animal {
  def manger(): Unit                           // Méthode abstraite
  def dormir(): Unit = println("Je dors")      // Méthode concrète
}

class Chien extends Animal {
  def manger(): Unit = println("Je mange des croquettes")
}

class Chat extends Animal {
  def manger(): Unit = println("Je mange du poisson")
}

val monChien = new Chien()
monChien.manger()  // Je mange des croquettes
monChien.dormir()  // Je dors
Caractéristiques des traits :
  • Méthodes abstraites : Doivent être implémentées par les classes qui étendent le trait
  • Méthodes concrètes : Déjà  définies dans le trait, peuvent être utilisées directement
  • Composition flexible : Une classe peut étendre plusieurs traits
  • Réutilisabilité : Partage de comportements communs

Code à  analyser 
def multiply(x: Int, y: Int): Int = x * y
println(multiply(2, 3))
Réponse correcte : 6
Analyse étape par étape :
  1. Définition de la méthode multiply : Prend deux arguments x et y de type Int et retourne leur produit.
  2. Appel de la méthode : multiply(2, 3) calcule 2 * 3 = 6
  3. Affichage du résultat : println(6) affiche 6 dans la console
Pourquoi pas les autres réponses ?
  • 2 * 3 : C'est l'opération interne, pas la sortie
  • multiply(2,3) : C'est l'appel de fonction, pas le résultat affiché
  • Erreur de compilation : Le code est syntaxiquement correct

En Scala, foreach est la méthode la plus efficace et idiomatique pour itérer sur tous les éléments d'une liste et appliquer une fonction à  chaque élément.

Qu'est-ce qu'une for-comprehension ?

En Scala, une for-comprehension est une version plus puissante de la boucle for. Elle permet non seulement d'itérer sur des collections, mais aussi de retourner un résultat en construisant de nouvelles collections.

Exemple - For-comprehension avec yield 
val numbers = List(1, 2, 3, 4, 5)

// For-comprehension qui retourne un résultat
val doubledEvens = for {
  n <- numbers           // Prendre chaque élément
  if n % 2 == 0         // Filtrer les nombres pairs
} yield n * 2           // Doubler et retourner

// Résultat: List(4, 8)
println(doubledEvens)
Explication étape par étape :
  1. n <- numbers : Itère sur chaque élément de la liste
  2. if n % 2 == 0 : Filtre pour ne garder que les nombres pairs
  3. yield n * 2 : Double chaque nombre pair et l'ajoute à  la nouvelle collection
Avantages
  • Concision : Une syntaxe élégante pour des opérations complexes
  • Lisibilité : Plus facile à  lire que des chaînes de map/filter
  • Puissance : Combine filtrage, transformation et création de collections

Ce que fait yield :

Le mot-clé yield permet de retourner une nouvelle collection qui contient les résultats de chaque itération de la boucle.

Comparaison avec et sans yield 
val numbers = List(1, 2, 3, 4, 5)

// SANS yield - Juste affichage
for (n <- numbers) println(n * 2)
// Affiche: 2, 4, 6, 8, 10 (mais ne retourne rien)

// AVEC yield - Crée une nouvelle collection
val doubled = for (n <- numbers) yield n * 2
// Résultat: List(2, 4, 6, 8, 10)
Processus étape par étape :
  1. Parcours des éléments : Iterate sur chaque élément de la collection
  2. Transformation : Applique une opération à  chaque élément
  3. Création d'une nouvelle collection : yield stocke chaque résultat dans une nouvelle collection
Exemple avec filtre
val evensDoubled = for {
  n <- numbers
  if n % 2 == 0    // Filtre
} yield n * 2      // Transformation et collection

// Résultat: List(4, 8)

Les switch statements en C

Pourquoi cette similitude ?

Le pattern matching en Scala ressemble aux switch statements car les deux comparent une valeur à  différents motifs et exécutent du code selon la correspondance trouvée.

Comparaison Switch vs Pattern Matching 
// Switch en C
switch(number) {
  case 1: printf("Un"); break;
  case 2: printf("Deux"); break;
  default: printf("Autre");
}
// Pattern matching en Scala
number match {
  case 1 => println("Un")
  case 2 => println("Deux")
  case _ => println("Autre")
}
Mais le pattern matching est plus puissant !
Décomposition de structures complexes 
case class Person(name: String, age: Int)

val person = Person("Alice", 25)

person match {
  case Person(name, age) if age >= 18 => 
    println(s"$name est majeur(e)")
  case Person(name, _) => 
    println(s"$name est mineur(e)")
}
Avantages du Pattern Matching
  • Décomposition de structures
  • Guards (conditions)
  • Extraction de valeurs
  • Vérification d'exhaustivité
Limitations du Switch
  • Valeurs simples uniquement
  • Pas de décomposition
  • Risque d'oubli de break
  • Moins expressif

flatMap utilise une fonction qui retourne une collection, tandis que map utilise une fonction qui retourne un seul élément.

Exemple - map vs flatMap

Option est un type conteneur qui peut soit contenir une valeur (Some(valeur)), soit être vide (None). Il remplace avantageusement l'utilisation de null.

Problèmes avec null
  • NullPointerException imprévisibles
  • Oubli facile de vérifications
  • Code moins lisible
  • Erreurs à  l'exécution
Avantages d'Option
  • Vérification explicite obligatoire
  • Erreurs détectées à  la compilation
  • Code plus expressif
  • Intégration avec les méthodes fonctionnelles

Qu'est-ce que implicit ?

Le mot-clé implicit permet au compilateur Scala de passer automatiquement des paramètres ou d'effectuer des conversions sans que vous ayez à  les spécifier explicitement.

Principe du lazy val

Un lazy val retarde le calcul d'une valeur jusqu'à  ce qu'elle soit réellement nécessaire. C'est une optimisation intelligente !

Exemple - Calcul retardé 
// Déclaration lazy - PAS de calcul immédiat
lazy val expensiveCalculation: Int = {
  println("Calcul en cours...") // Ne s'exécute que si nécessaire
  Thread.sleep(1000) // Simulation d'un calcul coà»teux
  42
}

println("Avant d'utiliser lazy val")
// Aucun calcul n'a encore eu lieu

println(expensiveCalculation) // ðŸ”¥ ICI le calcul se fait
// Affiche: "Calcul en cours..." puis "42"

println(expensiveCalculation) // Réutilise la valeur déjà  calculée
// Affiche directement: "42" (pas de recalcul)
Cycle de vie d'un lazy val :
1. Déclaration

Variable créée mais valeur pas encore calculée

2. Premier accès

Calcul effectué et valeur mémorisée

3. Accès suivants

Valeur réutilisée directement

Cas d'usage typiques :
  • Calculs coà»teux : Opérations qui prennent du temps
  • Chargement de ressources : Fichiers, connexions base de données
  • Optimisation mémoire : éviter de charger des données inutiles
  • Initialisation conditionnelle : Calcul seulement si utilisé
Avantage clé

Si vous n'utilisez jamais la lazy val, le calcul ne sera jamais effectué, économisant ainsi des ressources précieuses !

Qu'est-ce qu'un singleton avec object ?

Un object en Scala crée automatiquement un singleton - une instance unique accessible globalement dans votre programme.

  • Pas de gestion d'instance
  • Accès direct aux méthodes
Sécurité

  • Thread-safe par défaut
  • Initialisation paresseuse automatique
