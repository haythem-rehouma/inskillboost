Module 2 - Mettre en place l'environnement pour Scala

Configurez un environnement de développement complet pour Scala et Apache Spark

Module 2

Introduction

Un environnement de développement bien configuré est la clé du succès pour programmer en Scala et utiliser Apache Spark. Ce module vous guidera étape par étape pour installer et configurer tous les outils nécessaires.

Ce que vous allez installer
  • Java Development Kit (JDK) : Base pour Scala
  • Scala : Le langage de programmation
  • sbt : Outil de build pour Scala
  • IDE : IntelliJ IDEA ou VS Code
  • Apache Spark : Framework Big Data
  • Plugins : Extensions pour l'IDE
  • Git : Contrôle de version
  • Outils de test : Validation de l'installation
Temps d'installation estimé

Prévoyez 1 à  2 heures pour une installation complète, selon votre système et votre connexion Internet. Suivez les étapes dans l'ordre pour éviter les problèmes de compatibilité.

Prérequis système

Windows
  • Windows 10/11
  • 8 GB RAM minimum
  • 5 GB espace disque
  • Droits administrateur
macOS
  • macOS 10.14+
  • 8 GB RAM minimum
  • 5 GB espace disque
  • Homebrew recommandé
Linux
  • Ubuntu 18.04+
  • 8 GB RAM minimum
  • 5 GB espace disque
  • Curl et wget installés
Checklist avant installation

Installation Java Development Kit (JDK)

Java JDK 11 ou 17 (Recommandé)

Scala s'exécute sur la JVM (Java Virtual Machine). Nous recommandons OpenJDK 11 ou 17 pour une compatibilité optimale avec Scala et Spark.

Windows
macOS
Linux
Installation sur Windows
Option 1: Téléchargement manuel 
# 1. Aller sur https://adoptium.net/
# 2. Choisir "Temurin 17 (LTS)"
# 3. Sélectionner Windows x64
# 4. Télécharger le fichier .msi
# 5. Exécuter l'installeur en tant qu'administrateur
Option 2: Avec Chocolatey 
# Installer Chocolatey si pas déjà  fait
Set-ExecutionPolicy Bypass -Scope Process -Force
iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))

# Installer OpenJDK 17
choco install openjdk17
Configuration des variables d'environnement
  1. Ouvrir "Paramètres système avancés"
  2. Cliquer sur "Variables d'environnement"
  3. Ajouter JAVA_HOME pointant vers le dossier d'installation JDK
  4. Ajouter %JAVA_HOME%\bin au PATH
Installation sur macOS
Avec Homebrew (Recommandé) 
# Installer Homebrew si pas déjà  fait
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Installer OpenJDK 17
brew install openjdk@17

# Créer un lien symbolique
sudo ln -sfn /usr/local/opt/openjdk@17/libexec/openjdk.jdk \
     /Library/Java/JavaVirtualMachines/openjdk-17.jdk
Configuration du profil shell 
# Ajouter à  ~/.zshrc ou ~/.bash_profile
echo 'export JAVA_HOME=/usr/local/opt/openjdk@17' >> ~/.zshrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.zshrc

# Recharger le profil
source ~/.zshrc
Installation sur Linux (Ubuntu/Debian)
Installation avec apt 
# Mettre à  jour la liste des paquets
sudo apt update

# Installer OpenJDK 17
sudo apt install openjdk-17-jdk

# Vérifier l'installation
java -version
javac -version
Configuration des variables d'environnement 
# Ajouter au fichier ~/.bashrc
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc

# Recharger le profil
source ~/.bashrc

Test de l'installation

Installation Apache Spark

Apache Spark 3.5.x

Installation d'Apache Spark pour le développement local. Nous utilisons la version pré-compilée pour Hadoop.

Téléchargement et installation 
# Télécharger Spark (remplacer par la version actuelle)
curl -O https://archive.apache.org/dist/spark/spark-3.5.0/spark-3.5.0-bin-hadoop3.tgz

# Extraire l'archive
tar -xzf spark-3.5.0-bin-hadoop3.tgz

# Déplacer vers un dossier standard
sudo mv spark-3.5.0-bin-hadoop3 /opt/spark
Windows
macOS/Linux
Configuration Windows 
# Ajouter aux variables d'environnement:
# SPARK_HOME = C:\opt\spark
# PATH += %SPARK_HOME%\bin

# Télécharger winutils.exe pour Hadoop:
# https://github.com/steveloughran/winutils/blob/master/hadoop-3.0.0/bin/winutils.exe
# Placer dans %SPARK_HOME%\bin\

Premier programme Spark 
// Dans votre projet sbt, ajouter au build.sbt:
libraryDependencies += "org.apache.spark" %% "spark-core" % "3.5.0"
libraryDependencies += "org.apache.spark" %% "spark-sql" % "3.5.0"

// Exemple de code Spark:
import org.apache.spark.sql.SparkSession

object SparkExample extends App {
  val spark = SparkSession.builder()
    .appName("Premier Test Spark")
    .master("local[*]")
    .getOrCreate()
    
  import spark.implicits._
  
  val data = Seq(("Alice", 25), ("Bob", 30), ("Charlie", 35))
  val df = data.toDF("nom", "age")
  
  df.show()
  df.filter($"age" > 28).show()
  
  spark.stop()
}

Guide de dépannage

Solutions
  1. Vérifier l'installation : Java est-il bien installé ?
  2. Variables d'environnement : JAVA_HOME défini ? PATH mis à  jour ?
  3. Redémarrer le terminal après modification des variables
  4. Réinstaller Java si nécessaire
Test de diagnostic
# Windows
where java
echo %JAVA_HOME%

# Unix
which java
echo $JAVA_HOME
/usr/libexec/java_home -V  # macOS uniquement

Pourquoi sbt est lent ?

Au premier lancement, sbt télécharge toutes ses dépendances et celles de Scala. Cela peut prendre plusieurs minutes selon votre connexion Internet.

Optimisations
  1. Patience : Le premier lancement prend du temps
  2. Garder sbt ouvert : Utiliser sbt shell
  3. Augmenter la mémoire : Créer .sbtopts
Fichier .sbtopts pour optimiser sbt
# Créer .sbtopts dans le dossier du projet
-Xmx2G
-XX:+UseG1GC
-XX:+CMSClassUnloadingEnabled

étapes de résolution
  1. Vérifier la version : Plugin compatible avec votre IntelliJ ?
  2. Redémarrer IntelliJ après installation du plugin
  3. Réimporter le projet : File â†’ Reload sbt project
  4. Vérifier le SDK : Project Structure â†’ Project SDK
  5. Nettoyer le cache : File â†’ Invalidate Caches and Restart
Problèmes connus
  • IntelliJ version trop ancienne
  • Conflit entre versions Java
  • Projet sbt corrompu

Diagnostics Spark
  1. Variables d'environnement : SPARK_HOME défini ?
  2. Java compatible : Java 11 ou 17 requis
  3. Mémoire suffisante : Minimum 4 GB RAM libre
  4. Ports disponibles : Port 4040 libre ?
Tests de diagnostic Spark
# Tester Spark avec logs détaillés
spark-shell --conf spark.ui.port=4041 --driver-memory 2g

# Vérifier les logs
tail -f $SPARK_HOME/logs/spark-*.out
Configuration minimale recommandée
  • 4 GB RAM libre minimum
  • Java 11 ou 17
  • Spark 3.3+ pour Java 17
  • Firewall autorisant les ports 4040-4050
